Contacting the Syncthing Team Securely

If you believe that you've found a Syncthing-related security vulnerability, please report it by sending an email to security@syncthing.net. Your report should be acknowledged withing 48 hours. The PGP key for security@syncthing.net (B683AD7B76CAB013) below can be used to send encrypted mail or to verify responses received from that address.

Being able to release a well tested fix in a coordinated manner is key to ensuring users' security. Please do not file the issue in the public bug tracker. Doing so makes the issue public knowledge and necessitates a mad scramble to create a fix and get users to upgrade. We will announce the details of the issue together with the release, with proper attribution to the finder.

Release Signatures

The PGP key for release@syncthing.net (D26E6ED000654A3E) can be used to verify the signatures of official binary releases. The key is used for releases newer than v0.10.14. For version v0.10.14 and earlier the key for jakob@nym.se (https://nym.se/gpg.txt, 49F5AEC0BCE524C7) was used. The new release key (release@syncthing.net) is signed by the old key (jakob@nym.se).

When performing automatic upgrades, Syncthing verifies the download using a compiled in elliptic curve signature.

Verifying a Release Signature

Download the release file you need, and the sha1sum.txt.asc file

$ curl -sLO https://github.com/syncthing/syncthing/releases/download/v0.10.14/syncthing-linux-amd64-v0.10.14.tar.gz
$ curl -sLO https://github.com/syncthing/syncthing/releases/download/v0.10.14/sha1sum.txt.asc

Verify that the SHA1 checksum is correct for the release. Errors will be
printed for the release files you did not download - these can be ignored. The
important line is the bolded one, that indicates the checksum is correct for
the downloaded release file.

$ sha1sum -c sha1sum.txt.asc
...
sha1sum: syncthing-linux-386-v0.10.14.tar.gz: No such file or directory
syncthing-linux-386-v0.10.14.tar.gz: FAILED open or read
syncthing-linux-amd64-v0.10.14.tar.gz: OK
sha1sum: syncthing-linux-armv5-v0.10.14.tar.gz: No such file or directory
syncthing-linux-armv5-v0.10.14.tar.gz: FAILED open or read
...
sha1sum: WARNING: 20 lines are improperly formatted
sha1sum: WARNING: 12 listed files could not be read

Import the release keys (only necessary if you haven't done this previously).

$ gpg --keyserver pool.sks-keyservers.net --recv-key 49F5AEC0BCE524C7 D26E6ED000654A3E
gpg: requesting key BCE524C7 from hkp server pool.sks-keyservers.net
gpg: requesting key 00654A3E from hkp server pool.sks-keyservers.net
gpg: key BCE524C7: public key "Jakob Borg (calmh) <jakob@nym.se>" imported
gpg: key 00654A3E: public key "Syncthing Release Management <release@syncthing.net>" imported
gpg: no ultimately trusted keys found
gpg: Total number processed: 2
gpg:               imported: 2  (RSA: 2)

Verify the signature on the checksum file. Again, the bolded line is the important one.

$ gpg --verify sha1sum.txt.asc
gpg: Signature made Mon 29 Dec 2014 09:51:39 AM CET using RSA key ID BCE524C7
gpg: Good signature from "Syncthing Release Management <release@syncthing.net>"
gpg: WARNING: This key is not certified with a trusted signature!
gpg:          There is no indication that the signature belongs to the owner.
Primary key fingerprint: 37C8 4554 E7E0 A261 E4F7  6E1E D26E 6ED0 0065 4A3E
